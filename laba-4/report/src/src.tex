\section{Описание}
Требуется реализовать поиск образца-маски в тексте, состоящем из чисел в диапазоне от $0$ до $2^{32}-1$.
Символ \enquote{?} выступает в роли джокера и может совпадать с любым числом.
Согласно \cite{AhoCorasick} базовый алгоритм Ахо-Корасик позволяет найти все вхождения конечного множества образцов в текст за линейное время.
При наличии джокеров образец разбивается на несколько фрагментов, не содержащих джокеров.
Для каждого фрагмента запоминается его позиция в исходном образце.
Затем все фрагменты ищутся одновременно при помощи автомата Ахо-Корасик.
Когда некоторый фрагмент найден, вычисляется возможное начало полного образца.
Если все фрагменты одного образца указывают на одно и то же начало, образец считается найденным.
Образец, состоящий только из джокеров, обрабатывается отдельно: перечисляются все допустимые начальные позиции в тексте.

\section{Исходный код}
Программа написана на языке C++17. Входные данные поступают через стандартный поток ввода и представляют собой последовательность чисел (разделённых пробелами или переводами строк).
Первая строка содержит образец-маску, остальные строки – текст.

Структура \texttt{Symbol} хранит значение числа и логический признак джокера. Символы образца объединяются в класс \texttt{Pattern}, который умеет читать поток и строить последовательность из чисел и джокеров.

Центральная часть алгоритма – класс \texttt{Trie}, реализующий бор фрагментов и автомат Ахо-Корасик.
Узел бора представлен структурой \texttt{Node}:
\begin{lstlisting}[language=C++]
struct Node {
    std::unordered_map<unsigned int, Node*> next;
    std::vector<FragmentInfo> fragments;
    Node* fail = nullptr;
    Node* out = nullptr;
    // ...
};
\end{lstlisting}
Ключевая особенность – использование \texttt{unordered\_map} для переходов, поскольку размер алфавита $2^{32}$ не позволяет применить обычный массив.
В каждом терминальном узле хранится вектор \texttt{FragmentInfo}, содержащий номер образца, конечную позицию фрагмента в образце и его длину.

При построении бора образец разбивается на непрерывные цепочки чисел (без джокеров). Каждый такой фрагмент вставляется в бор, и в последней вершине прописывается информация о нём.
После вставки всех фрагментов вызывается метод \texttt{buildLinks}, который с помощью BFS строит суффиксные ссылки (\texttt{fail}) и сжатые выходные ссылки (\texttt{out}).

Сканирование текста выполняет класс \texttt{Scanner}. Он поддерживает текущую позицию в автомате, абсолютную позицию в тексте и набор частичных совпадений.
Метод \texttt{feedSymbol} перемещает автомат по входному символу и для каждой встреченной терминальной вершины (как текущей, так и достижимой по \texttt{out}) вызывает \texttt{saveMatch}.
В \texttt{saveMatch} вычисляется потенциальное начало образца \texttt{abs\_start} и обновляется вектор \texttt{Match} – для каждого образца хранятся кандидаты на начало и счётчик голосов (число совпавших фрагментов, указавших на это начало).

После обработки всего текста метод \texttt{getResults} оставляет только те совпадения, где количество голосов равно общему числу фрагментов образца, и где образец полностью умещается в пределах прочитанного текста.
Дополнительно пересчитываются линейные и столбцовые координаты.

Класс \texttt{PatternMatcher} объединяет бора и сканер, а также выделяет особые образцы, состоящие исключительно из джокеров. Для них результат генерируется прямым перебором всех допустимых начальных позиций.
